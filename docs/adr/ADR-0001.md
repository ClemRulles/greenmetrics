# ADR-0001: API Style (REST + OpenAPI) vs tRPC
- **Status**: Proposed
- **Context**: MVP needs interoperability (partner exports, future supplier portal), typed clients, and external docs.
- **Decision**: Use **REST** with **OpenAPI 3** contract. Keep tRPC as a future option for internal RPC if needed.
- **Consequences**:
  - ✅ Clear external interface, easy SDK generation, Redoc/Swagger docs.
  - ✅ Stable contracts help partners and compliance reviews.
  - ❌ Slightly more boilerplate than tRPC.
- **Alternatives**: tRPC (fast dev but less friendly for external consumers), GraphQL (overkill for MVP).
