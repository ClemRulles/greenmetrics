# GreenMetrics Report Specification

## Overview

This specification defines the structure, content, and metadata requirements for PDF reports generated by GreenMetrics. The primary framework is **VSME Basic** with optional VSME Comprehensive fields and mapping hooks to ESRS 2 (general disclosures) and minimal E1 (Scopes 1–2).

## Framework Targets

- **Primary**: VSME Basic (Value Sustainability Management for SMEs - Basic Level)
- **Optional**: VSME Comprehensive fields (marked clearly)
- **Mapping**: ESRS 2 (general disclosures) and E1 (climate change) minimum requirements
- **Scope**: Scopes 1–2 emissions only (MVP boundary)

## Report Metadata Schema

All reports must persist the following metadata:

```typescript
interface ReportMetadata {
  framework: string;           // "VSME_BASIC" | "VSME_COMPREHENSIVE" | "ESRS"
  frameworkVersion: string;    // "1.0", "2.1", etc.
  factorsVersion: string;      // "ADEME_BC_23.0", "EPA_2024", etc.
  computationMethod: string;   // "ACTIVITY_BASED" | "SPEND_BASED"
  reportingPeriod: {
    startDate: string;         // ISO 8601 date
    endDate: string;           // ISO 8601 date
  };
  orgGeography: string;        // ISO 3166-1 alpha-2 country code
  language: string;            // "en" | "fr"
  renderEngine: string;        // "REACT_PDF" | "PUPPETEER"
  generatedAt: string;         // ISO 8601 timestamp
}
```

## Acceptance Criteria

### Mandatory Sections Checklist

- [ ] **Cover Page**: Organization name, period, framework clearly stated (EN/FR)
- [ ] **Management Statement**: Leadership commitment present (EN/FR)  
- [ ] **Company Profile**: Industry, size, geography identified (EN/FR)
- [ ] **Reporting Boundary**: Scope definition explicit (EN/FR)
- [ ] **Methodology**: Factor sources, versions, geography specified (EN/FR)
- [ ] **Activity Data**: Input quantities with units listed (EN/FR)
- [ ] **Results**: Scope 1 and Scope 2 totals calculated (EN/FR)
- [ ] **YoY Comparison**: Previous period comparison if available (EN/FR)
- [ ] **Assurance Note**: Third-party verification status (EN/FR)
- [ ] **Glossary**: Key terms defined (EN/FR)

### Technical Requirements

- [ ] **Scopes 1–2**: Both scopes present and calculated
- [ ] **Factor Provenance**: Source, version, geography explicit for all factors
- [ ] **Metadata**: All required metadata fields populated
- [ ] **Traceability**: Each result links to specific factor version (see factors-and-traceability.md)
- [ ] **Bilingual**: All sections available in EN and FR
- [ ] **Framework Alignment**: VSME Basic requirements covered
- [ ] **Optional Fields**: VSME Comprehensive and ESRS items clearly marked

## Assumptions

- **Assumption**: SMEs primarily need VSME Basic; ESRS mapping supports larger clients transitioning to CSRD
- **Assumption**: Factor updates occur annually; version tracking prevents calculation drift
- **Assumption**: PDF format sufficient for audit requirements; structured data export available separately
- **Assumption**: French market focus justifies EN/FR bilingual priority over other languages
